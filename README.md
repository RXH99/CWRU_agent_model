### W3：小样本改造 → 原型网络（读研核心）

#### 为什么从「迁移学习」换成「原型网络」

| 方案 | 效果 | 学习成本 |
|------|------|---------|
| 迁移学习/微调 | 简单直接，但小样本下效果不稳定 | ⭐⭐ |
| **原型网络** | 小样本任务的标准方法，每类 5 样本也能稳定出效果 | ⭐⭐⭐⭐ |

原型网络的核心思想：**不学"分类边界"，学"特征空间中的距离比较"。**

```
每类 5 个样本 → 算 5 个"类原型"（Prototype）
待分类样本 → 映射到同一个特征空间
看它离哪个类原型最近 → 判给哪一类
```

**为什么适合故障诊断：**
- 工业场景故障数据天然稀缺
- 出现新故障类型时，拿几个新样本现场算新原型就能扩展，不用重训

#### 具体任务

**第一步：用 W2 的 1D-CNN 提取特征**
- 去掉最后的全连接层和 Softmax，把 CNN 作为特征提取器（Encoder）
- 输入信号 → Encoder → 特征向量

**第二步：实现原型网络逻辑**
- 支持集（Support Set）：每类随机抽 5 个样本，Encoder 输出取均值 → 类原型
- 查询集（Query Set）：待分类样本，Encoder 输出 → 算距离（欧氏距离）
- 损失函数：交叉熵，在距离上做 Softmax

**第三步：Episodic Training**
- 每个 Episode：随机选 N 个类，每类 K 个 Support + 若干 Query
- 在这个 Episode 内计算损失、更新梯度
- 这样模型学会的是"如何比较"，而不是"记住类别"

**第四步：对比实验**

| 方法 | 准确率 | 每类样本数 |
|------|--------|-----------|
| W2 全量 1D-CNN（baseline） | xx% | 大量 |
| 全量数据下的小样本模拟 | xx% | 5 条 |
| 原型网络（5-way 5-shot） | xx% >80% | 5 条 |
| 原型网络 + 数据增强 | xx% | 5 条 |

**数据增强手段：**
- 加高斯噪声
- 时间平移（随机偏移起始点）
- 随机裁剪

**做什么：**
1. 实现 1D-Grad-CAM（一维信号的 Grad-CAM 变体）
2. 可视化：

**核心价值：** 证明模型不是"瞎猜"的——热力图能看出模型关注了信号中物理上合理的故障特征。

**架构：**
```
用户输入
  ↓
LangChain Agent（Ollama/Qwen 作为推理引擎）
  ↓
┌────┬────┬────┬──────┐
│T1  │T2  │T3  │  T4   │   ← 工具调用
└─┬──┴─┬──┴─┬──┴──┬───┘
  │    │    │     │
  ↓    ↓    ↓     ↓
诊断  解释  知识库  报告生成
模型  热力图  检索
```

**四个 Tool：**

| Tool | 功能 | 技术实现 |
|------|------|---------|
| T1: diagnose | 调用原型网络模型推理 | 加载预训练权重 → 预测故障类别 + 置信度 |
| T2: explain | 生成 Grad-CAM 热力图 | 调用 W4 的代码，返回图片 base64 |
| T3: knowledge | 查询维修建议 | LangChain + ChromaDB 检索知识库 |
| T4: report | 生成综合诊断报告 | LLM 整合 T1/T2/T3 结果 |

**Agent 工作流程：**
1. 用户说：「帮我诊断这段轴承数据」
2. Agent → 调用 diagnose → 得到故障类型 + 置信度
3. Agent → 调用 explain → 得到热力图可视化
4. Agent → 调用 knowledge → 检索对应维修建议
5. Agent → 调用 report → 整合输出完整诊断报告

**进阶：对话上下文管理**
- Agent 能记住历史对话
- 用户可以对上一次诊断结果追问："为什么不是外圈故障？"
- Agent 能对比分析多段信号
